## Namecheap 虚拟卡付款失败？按这个顺序排查

绑卡 / 付款失败基本就这几类原因，按概率从高到低排查：

**1. 余额不够**

Namecheap 结算时除了域名本身的价格，会加一笔 0.18 美元的 ICANN 注册费。如果你卡里余额刚好只够域名价格，结算会直接失败。**建议卡内至少多留 1–2 USDT 的 buffer。**

**2. 卡段被风控**

不是所有虚拟卡 BIN 都能过 Namecheap 的风控。PokePay 的 5347 Visa 和 559666 MasterCard 白金这两个 BIN 在 Namecheap 上实测通过率较高。如果你试的是某个新 BIN 或冷门 BIN 一直被拒，换个卡段试试。

**3. 3DS 验证没过**

PokePay 的 Visa 卡段支持 3D Secure。如果 Namecheap 弹出 3DS 验证页面，正常情况下 PokePay App 会有验证通知，按提示完成就行。如果 3DS 一直验证不了，检查一下 App 是不是没开验证通知权限。

**4. 账单地址和 IP 不匹配**

Namecheap 的支付网关会做账单地址 + IP 地理位置的一致性校验。你填的账单地址国家，最好跟你访问 IP 所在国家一致。比如你挂了美国 IP，账单地址就填美国地址，别填香港。

**5. 短时间内频繁试卡**

如果你同一个订单 5 分钟内连续试了三四次都失败，Namecheap 会临时锁定这张卡的支付尝试。**别一直戳，等 1–2 小时再试，或者换张卡。**

**6. 不被 Namecheap 接受的卡组织**

PokePay 全部走 Visa 和 MasterCard 两个组织，Namecheap 都接受。这个不用太担心。但如果哪天 Namecheap 改了支付策略，以它结算页实时显示的可用支付方式为准。
